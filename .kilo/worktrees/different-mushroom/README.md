# ИИ-ассистент для получения учёного звания — ИТМО

Веб-приложение для автоматизированной проверки соответствия критериям учёного звания (доцент/профессор) согласно Постановлению Правительства РФ №1139. Разработано для Университета ИТМО.

---

## Возможности

- **Проверка критериев** — автоматическая проверка данных соискателя по актуальным критериям ВАК
- **ИИ-чат** — ответы на вопросы по нормативной базе в режиме реального времени
- **История заявок** — сохранение и просмотр всех поданных заявок
- **Загрузка документов** — прикрепление справок и документов к заявке
- **Экспертная панель** — ручная проверка заявок с комментариями
- **Административная панель** — управление критериями, пользователями, статистика
- **Роли пользователей** — соискатель, эксперт, администратор

## Установка и запуск (Windows)

### Шаг 1. Установка Node.js

1. Перейдите на https://nodejs.org и скачайте LTS-версию (20.x)
2. Запустите установщик, отметьте "Automatically install necessary tools"
3. Откройте **PowerShell** и проверьте установку:
   ```powershell
   node --version   # должно показать v20.x.x
   npm --version    # должно показать 10.x.x
   ```

### Шаг 2. Установка pnpm

```powershell
npm install -g pnpm
pnpm --version   # должно показать 8.x.x или выше
```

### Шаг 3. Установка PostgreSQL

1. Скачайте PostgreSQL с https://www.postgresql.org/download/windows/
2. Запустите установщик, запомните пароль для пользователя `postgres`
3. В установщике убедитесь, что выбраны: PostgreSQL Server, pgAdmin 4, Command Line Tools
4. Создайте базу данных через pgAdmin 4 или командную строку:
   ```powershell
   # Откройте командную строку PostgreSQL (psql)
   psql -U postgres
   # Создайте базу данных
   CREATE DATABASE itmo_academic_rank;
   # Выйдите
   \q
   ```

### Шаг 4. Клонирование и установка зависимостей

```powershell
# Клонируйте репозиторий
git clone <URL репозитория>
cd <папка проекта>

# Установите все зависимости
pnpm install
```

### Шаг 5. Настройка переменных окружения

Создайте файл `.env` в корне проекта со следующим содержимым:

```env
# База данных PostgreSQL
DATABASE_URL=postgresql://postgres:ВАШ_ПАРОЛЬ@localhost:5432/itmo_academic_rank
PGHOST=localhost
PGPORT=5432
PGUSER=postgres
PGPASSWORD=ВАШ_ПАРОЛЬ
PGDATABASE=itmo_academic_rank

# Секретный ключ для JWT токенов (измените на случайную строку!)
JWT_SECRET=ваш-секретный-ключ-минимум-32-символа

# Ключ API для OpenAI (опционально — если запускаете без Replit)
# Получите на https://platform.openai.com/api-keys
OPENAI_API_KEY=sk-...
AI_INTEGRATIONS_OPENAI_BASE_URL=https://api.openai.com/v1
AI_INTEGRATIONS_OPENAI_API_KEY=ваш-ключ-openai
```

**Примечание:** Если запускаете на Replit, переменные `AI_INTEGRATIONS_OPENAI_*` устанавливаются автоматически и бесплатно.

### Шаг 6. Создание таблиц в базе данных

```powershell
pnpm --filter @workspace/db run push
```

### Шаг 7. Заполнение начальными данными

```powershell
pnpm --filter @workspace/scripts run seed
```

После выполнения будут созданы тестовые аккаунты:
- **Администратор:** admin@itmo.ru / admin123
- **Соискатель:** ivanov@itmo.ru / test123

### Шаг 8. Запуск приложения

**Бэкенд (в одном окне PowerShell):**
```powershell
pnpm --filter @workspace/api-server run dev
# Сервер запустится на http://localhost:8080
```

**Фронтенд (в другом окне PowerShell):**
```powershell
pnpm --filter @workspace/itmo-academic-rank run dev
# Приложение откроется на http://localhost:5173
```

Откройте браузер и перейдите на http://localhost:5173

## Критерии учёного звания

### Доцент (согласно Постановлению РФ №1139)
| Критерий | Требование |
|----------|-----------|
| Стаж научно-педагогической работы | ≥ 5 лет |
| Стаж педагогической работы по специальности | ≥ 3 лет |
| Публикации в рецензируемых изданиях | ≥ 10 за 5 лет |
| Учебные издания | ≥ 2 за 3 года |
| Публикации в Scopus/WoS | ≥ 2 |

| Критерий | Требование |

## Управление критериями

Администратор может редактировать критерии через административную панель. Каждое изменение создаёт новую версию — история изменений сохраняется.

## Часто задаваемые вопросы

**Как подключить свой OpenAI ключ?**
Укажите `AI_INTEGRATIONS_OPENAI_API_KEY` и `AI_INTEGRATIONS_OPENAI_BASE_URL=https://api.openai.com/v1` в `.env` файле.

**Как сделать пользователя администратором?**
Войдите как admin@itmo.ru и перейдите в раздел "Управление пользователями" в административной панели.

**Как обновить критерии?**
Войдите как администратор → Административная панель → Критерии → Редактировать.

**Как добавить нового эксперта?**
Пользователь регистрируется как обычный соискатель, затем администратор меняет его роль на "expert" через панель управления пользователями.
