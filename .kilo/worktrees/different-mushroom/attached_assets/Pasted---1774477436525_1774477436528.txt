Создай полноценное веб-приложение для ИТМО — ИИ-ассистент для проверки соответствия критериям учёного звания (доцент/профессор). Это реальный инструмент, который будет использоваться в университете, поэтому качество критически важно.

Бизнес-ценность: Автоматизация предварительной проверки заявок на учёные звания, сокращение времени обработки с нескольких дней до 5 минут, снижение нагрузки на отдел кадров.

Технические требования:

Backend (FastAPI + PostgreSQL)
Модуль аутентификации:

Регистрация (email, ФИО, должность, подразделение)

Вход (JWT токены, refresh токены)

Роли: applicant, expert, admin

Защита эндпоинтов через зависимости

Модуль критериев:

Критерии хранятся в БД (таблица criteria_rules)

Поддержка версионирования (история изменений)

Админ может редактировать критерии через API (JSON-структура)

Критерии для доцента и профессора (согласно Постановлению Правительства РФ №1139):

Доцент:

Стаж научно-педагогической работы: ≥ 5 лет

Стаж педагогической работы по специальности: ≥ 3 лет

Публикации в рецензируемых изданиях: ≥ 10 за 5 лет

Учебные издания: ≥ 2 за 3 года

Публикации в Scopus/WoS: ≥ 2

Учёная степень: кандидат наук

Профессор:

Стаж научно-педагогической работы: ≥ 10 лет

Стаж педагогической работы по специальности: ≥ 5 лет

Публикации в рецензируемых изданиях: ≥ 20 за 5 лет

Учебные издания: ≥ 3 за 5 лет

Публикации в Scopus/WoS: ≥ 5

Учёная степень: доктор наук

Подготовка аспирантов: ≥ 1 защитившегося

Модуль проверки:

Принимает структурированные данные от соискателя

Сопоставляет с актуальной версией критериев

Возвращает детализированный результат:

json
{
  "status": "partial",
  "score": 85,
  "criteria_breakdown": [
    {"name": "pedagogical_experience", "required": 3, "actual": 5, "met": true},
    {"name": "publications", "required": 10, "actual": 7, "met": false, "shortage": 3}
  ],
  "recommendations": ["Необходимо опубликовать ещё 3 статьи в рецензируемых журналах"],
  "missing_documents": ["Справка о стаже", "Список публикаций"]
}
Сохраняет результат в applications с привязкой к пользователю

Контекстный промпт с подгрузкой актуальных критериев из БД

База данных (PostgreSQL)
sql
-- Пользователи
users (id, email, full_name, department, position, role, is_active, created_at)

-- Критерии (версионирование)
criteria_rules (id, rank_type, rules_json, version, is_active, created_by, created_at)

-- Заявки
applications (id, user_id, rank_type, structured_data_json, result_json, status, expert_comment, created_at, updated_at)

-- Публикации (детализация)
publications (id, application_id, title, journal, year, type, pages, doi)

-- Документы
documents (id, application_id, file_name, file_url, file_type, uploaded_at)

-- Чат-история
chat_messages (id, user_id, message, response, context_app_id, rating, created_at)

-- Аудит
audit_logs (id, user_id, action, details, ip_address, created_at)
